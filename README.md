# サービス名
SokoNote ソコノート

「もっと安く買えたかも」を防ぐ底値×単価記録アプリ

# サービス概要
本サービスは、日々の食材や日用品の買い物において「以前より高く買ってしまう」失敗を防ぐための、底値管理・比較Webアプリです。

ユーザーは商品ごとに過去の購入価格を登録・保存でき、再度購入する際に現在価格と履歴データを即座に比較することができます。

また、内容量やパック数が異なる同一商品の単価を自動計算し、どちらの商品が最もお得か迷わず判断できます。

さらに将来的にVision APIを用いてチラシから価格情報を読み取り、登録済みデータとの比較結果を一覧で表示することで、買い物中でも素早く最適な判断をサポートすることを構想しております。

# 開発背景
私は買い物と節約が好きで、スーパーごとの価格や特徴を比較し、少しでも安く購入できたときに喜びを感じてきました。

しかしその一方で、

「別の店の方が安かった」

「以前はもっと安く買えた」

と後から気づく経験も多くありました。


日常の買い物では、同じ商品であっても店舗・時期・内容量によって価格が変動します。

そのため、どの店舗でどの量をいくらで購入したのかを記憶に頼って判断していましたが、

情報量が多く、人の記憶だけでは正確な比較に限界がありました。

また、チラシの「特価」という表現に影響され、実際にはそれほどお得ではない商品を購入してしまうこともありました。

メモアプリや家計簿アプリでの管理も試みましたが、買い物中に素早く価格比較を行うことは難しく、継続的に活用できませんでした。


そこで、過去の購入価格や内容量を商品ごとにデータとして蓄積し、現在の価格と自動的に比較することで、感覚ではなく根拠を持って価格判断ができる仕組みが必要だと考えました。

人が記憶や計算を行うのではなく、アプリ側が単価計算や底値比較を行うことで、買い物中でも迷わず意思決定できることを目指しています。

また、実際の買い物シーンではスマートフォンでの利用が前提となるため、Webアプリとして提供し、場所を選ばず即座にアクセスできるようにしました。

さらに将来的には、チラシを撮影するだけで価格情報を取得し、購入履歴と比較できる仕組みを取り入れることで、入力の手間を減らし、継続的に使えるサービスを構想しました。

# ターゲット層&理由
### メインターゲットユーザー
  **20代後半〜40代　共働きの主婦（主夫）層**

  - 週2〜3回は買い物をする
  - 節約意識があり、少しでも安く買いたいと考えている
  - チラシをよく見る習慣がある
  - スマートフォンでの情報管理は抵抗がない
  
  **選定理由**
  
  日常的に価格変動を体感しており、家族構成によるが購入量が多く、単価比較の効果を実感しやすい。

  また、共働きで時間が限られているため初回利用時は商品登録が必要ですが、一度登録すれば次回以降は即座に比較できるため、買い物中の素早い判断できるというニーズに着目しました。
    
### サブターゲットユーザー
  **20代〜30代の一人暮らしの会社員**

  - 自炊により食費を抑えたいと思っている
  - 時間がないため効率重視
  - いくお店はいつも同じ店が多い
  - 損をしたくないという気持ちがある
  
  **選定理由**

  内容量やパック数が異なる商品の選択がそのまま生活費・食費に直結しますが、家族を持つ層と比べると記録データや蓄積スピードは緩やかです。

  そのためメインターゲットではないものの、「損をしたくない」「効率よく判断したい」という価値提供としてサブターゲットにしました。

# ユーザー獲得方法（本リリース時実施）
本サービスは、日常的な買い物シーンでの利用を想定し、主婦（夫）層の利用率が高いSNS "LINE" を主要な導線として設計しました。

- LINE公式アカウントを起点としたUX設計
  - Webアプリ専用の公式アカウント作成
  - 友達追加時にアプリURLを自動送信して即アクセス可能
  - LINEログインによりユーザー登録の手間を最小限に削減
  - 同時にLINE友達追加を促しMessaging APIで自動あいさつメッセージを送信
  - 以降Webアプリの状態に応じて通知し、定期的な利用促進

これにより、「登録が面倒で使わなくなる」「存在を忘れてしまう」といったWebサービス特有の離脱を防ぎ、継続的に利用される体験設計を重視しています。

また、サービスの認知拡大を目的として、主婦（夫）層に向けた体験談・節約視点の記事をnoteにて発信することを想定しています。
　
- Webメディア（note）を活用した認知獲得
  - ストーリー形式による価値訴求
  - ターゲット層が共感できる「買い物失敗談」や「節約の成功体験」をベースとした記事の公開
  - 「底値管理」「単価比較」といったキーワードに関心を持つ層へ
  - 記事を通じたアプリの存在を認知させ、LINE公式アカウントへの流入経路を確保
  - 開発者の想いやアプリの使い方を丁寧に言語化し発信
  - 利用開始時の心理的ハードルを下げ、信頼性の構築によるファン（継続利用者）の獲得

これにより、サービスを必要とするユーザーに無理なく認知してもらい、継続的に利用される関係性の構築を目指しています。

# サービス利用のイメージ

  1. 初回利用：購入商品の価格を記録する

ユーザーは、スーパーで購入した商品をアプリに登録します。

商品名・メーカー・カテゴリ・店舗名・内容量・価格・単位（g / ml / 個数など）・購入日を入力することで、**自動的に単価（1g あたり、1個あたりなど）が計算されます。**

これにより、「どの店で」「どの量を」「いくらで買ったのか」という情報が、記憶ではなくデータとして蓄積されていきます。

  2. 買い物前・買い物中：価格の妥当性を即確認

次に同じ商品を購入しようとした際、商品を選択し現在の価格を入力するだけで、以下の情報が即座に確認できます。

過去に購入した最安値（底値）との比較（高い / 安い）両方の単位ごとの単価（@円/100mlあたり）

これにより、「今日は安いのか？」「今買うべきか、見送るべきか？」を感覚ではなく、根拠を持って判断できるようになります。

  3. 同一商品の単価比較

同じ商品であっても、内容量やパック数が異なる商品が並んでいることは珍しくありません。

内容量（g / ml / 個数）をもとに単価を自動計算することで、条件の異なる同一商品を同じ基準で比較できます。

例えば、

  700ml：〇〇円（¥◯◯ / 100ml）

  1,200ml：〇〇円（¥◯◯ / 100ml）

  2個パック：〇〇円（¥◯◯ / 個）

といった形で単価が一覧表示され、どの商品が本当にお得かを一目で把握できます。

これにより、「大容量だから安そう」「まとめ買いだからお得そう」といった印象に左右されず、数字に基づいた判断が可能になります。

  4. （将来機能）チラシ撮影による自動価格比較

将来的には、チラシや店頭価格を撮影するだけで、商品名・価格・内容量を自動で読み取り、登録済みの購入履歴と比較する機能を想定しています。

これにより、

- 手入力の手間を削減
- 買い物中でもワンタップで価格判断
- 継続的に使いやすいサービス体験

を実現することを目指しています。

  5. 利用後の変化

本サービスを利用することで、ユーザーは以下の変化を実感できます。    

- 「以前より高く買ってしまった」という後悔が減る
- 本当に安いタイミングを把握できる
- 買い物中の迷いが減り、時間短縮につながる
- 節約行動に納得感と楽しさが生まれる    

**記憶や感覚に頼らず、データに基づいた買い物判断**をサポートすることが、本アプリの提供価値です。

# 類似サービスとの差別化・推しポイント

- メモアプリ
  - 情報を自由に記録できる一方で、構造化されていないため検索や計算が難しい。結果として単価を出すには別途手動で行う必要がある。

- 家計簿アプリ
  - 「いくら使ったか（過去）」の記録が目的であり、「今買うべきか（未来）」の判断材料は限定的。
  
      レシートOCR機能（文字をデジタルに変換する技術）が搭載されたアプリもあるが、内容量や単価まで考慮した比較は難しい

- 「どちらがお得」スマホアプリ
  - その場でAとBの比較は可能だが、自分の過去の購入価格や最安値を基準とした比較は行えない。

- 「ソコネ」スマホアプリ
  - 過去の底値を記録し、次回購入時の参考情報として活用できる一方で、買い物中に同一商品の内容量違いを単価ベースで即比較する機能は限定的。

本サービスはこれらと比べて
- 自分が過去に購入した価格を基準とした比較
- 内容量の異なる商品でも単価を即座に自動計算
→ 今、目の前にあるA商品とB商品のどちらを買うべきか、かつ、それは自分の過去の底値と比べてどうなのかという過去の商品×目の前の商品を同時に行う
- チラシ撮影だけで価格情報を取得できるvision APIを活用による入力負担の軽減を実現しています。
  
単なる比較ツールだけではなく、日常的な節約行動を変える体験設計を重視している点が本サービスの特徴です。
